Wie wird ein Canvas mit einem Use Case verknüpft?

Wo entsteht die Verbindung zwischen Canvas und Scoring-Eintrag?

A

Push vom Canvas

Im Canvas gibt es einen "In Scoring aufnehmen"-Button. Dieser erzeugt automatisch einen Scoring-Eintrag mit vorausgefüllten Werten.

Canvas — "OCR im Finanzwesen" · Ergebnis-Ansicht
Problem: Manuelle Rechnungsverarbeitung, 3 FTE gebunden
Lösung: OCR + SAP Document Intelligence
Branche: Finance · Backend: SAP S/4HANA
PDF Export
✚ In Scoring aufnehmen →

Für

  • Canvas-Felder werden direkt in Scoring übertragen
  • Eindeutige Richtung: Canvas → Scoring

Gegen

  • Kein nachträgliches Verknüpfen möglich
B

Pull im Scoring  Empfohlen

Jeder Use-Case-Eintrag im Scoring hat ein optionales "Canvas verknüpfen"-Dropdown. Bestehende Canvases sind auswählbar.

Use-Case Scoring — Eintrag bearbeiten
Name
Branche / Domäne
Canvas verknüpfen (optional)
◧ OCR im Finanzwesen (Canvas) ▾
✓ Canvas verknüpft — Branche, Technologiekontext und DSGVO-Anforderungen werden im Architektur-Generator berücksichtigt.

Für

  • Flexibel: Canvas nachträglich wählbar
  • Ein Scoring-Eintrag = ein Canvas (klare 1:1-Relation)
  • Keine Canvas-Pflicht, optionale Vertiefung

Gegen

  • Nutzer muss aktiv verknüpfen
C

Beides — bidirektional

Push vom Canvas UND Pull im Scoring. Beide Wege möglich, Verknüpfung immer sichtbar.

Verknüpfungs-Indikator (überall sichtbar)
◧ Canvas
OCR im Finanzwesen
⇄ Scoring: OCR Rechnung
⇄
⊞ Scoring-Eintrag
OCR Rechnungsverarb.
⇄ Canvas: OCR im Finanzw.

Für

  • Maximale Flexibilität
  • Überall sichtbarer Kontext

Gegen

  • Komplexerer Umbau
  • Zwei Wege = mehr Fehlerpotenzial
